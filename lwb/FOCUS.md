# LWB Focus Lists

These lists guide which concepts get surfaced in the 🧠 Learn section.
Primary focus is weighted highest. Secondary focus is next. Anything outside both is lowest priority.

---

## 🎯 Primary Focus

Concepts to prioritize above all others.

- Frontend engineering (React, TypeScript, component architecture, state management, rendering, CSS)
- Browser fundamentals (DOM, events, rendering pipeline, layout, performance, accessibility)
- JavaScript & TypeScript deep understanding (async behavior, closures, prototypes, type systems)
- Frontend system design, scalability, maintainability, and engineering trade-offs
- Interview readiness for SDE-2, Mid-Level, and Senior Frontend Engineer roles

---

## 🔁 Secondary Focus

Surface these when the code clearly touches these domains.

- Full Stack (API design, server-client data flow, auth, caching strategies)
- Node.js / backend patterns (middleware, streaming, queues)

---

## How to update

Edit this file directly to add, remove, or reprioritize topics.
Each bullet is a domain — keep descriptions short so the skill can match them to concepts.
